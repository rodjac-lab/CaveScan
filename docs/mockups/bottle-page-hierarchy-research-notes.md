# Hierarchie de fiche bouteille - notes de recherche

## Question

Pour un amateur eclaire en progression, avec une cave deja construite, quelles informations doivent remonter dans la fiche bouteille ?

## Ce que les sources pointent

- L'identite du vin reste le premier besoin : producteur, cuvee, appellation, millesime.
- `grape variety` / assemblage est un signal tres important pour comprendre le style.
- Les descriptions sensorielles aident fortement a la decision, souvent plus qu'un simple nom de cepage.
- La region compte, surtout quand l'utilisateur sait deja en tirer du sens.
- Le pays seul est rarement un signal de premier rang pour un amateur implique.
- Les informations de service et d'accord ont une vraie valeur d'usage.
- Les details techniques tres experts sont utiles plus bas dans la fiche, pas dans le hero.

## Criteres d'evaluation proposes

### 1. Orientation immediate

La fiche doit aider a repondre vite a :

- Qu'est-ce que c'est ?
- Quel style m'attend ?
- Qu'est-ce que j'en fais ce soir ?

### 2. Pouvoir predictif

Une bonne fiche doit permettre d'anticiper :

- structure
- tension / ampleur
- maturite
- usage a table

### 3. Valeur pedagogique

L'utilisateur doit apprendre sans effort la relation entre :

- appellation / region -> style
- cepages -> profil
- millesime -> maturite
- style -> service et accord

### 4. Non-redondance

Les champs qui repetent une information deja implicite doivent descendre.

Exemple :

- `France` ajoute peu si `Champagne` ou `Margaux` est deja visible.

### 5. Respect du role de CaveScan

La fiche n'est ni une fiche produit e-commerce, ni une encyclopedie.
Elle doit combiner :

- identite du vin
- aide a l'ouverture / au service
- contexte cave

## Lecture critique des 4 propositions

### Proposition A - Assemblage d'abord

Points forts :

- remonte une information vraiment utile
- bonne lecture pour les assemblages
- tres bonne valeur pedagogique

Points faibles :

- risque de sous-valoriser l'appellation si l'utilisateur pense d'abord "terroir"

### Proposition B - Region d'abord

Points forts :

- logique forte pour les amateurs deja orientes terroir
- bonne coherence avec la culture vin classique

Points faibles :

- plus faible aide immediate a la decision
- l'utilisateur doit deja savoir lire la region

### Proposition C - Decision d'abord

Points forts :

- excellente utilite pratique
- tres alignee avec l'usage CaveScan
- forte proximite avec le role "sommelier"

Points faibles :

- peut sembler moins "fiche bouteille" et plus "assistant"
- a bien calibrer pour ne pas noyer l'identite du vin

### Proposition D - Apprentissage guide

Points forts :

- combine identite, style, service et apprentissage
- correspond bien au ton Celestin : utile, cultivant, pas professoral
- remonte les cepages sans perdre l'appellation

Points faibles :

- demande plus de discipline de contenu
- repose sur la qualite des enrichissements de style et d'accord

## Recommandation provisoire

La piste la plus prometteuse pour CaveScan me semble etre un hybride entre A et C, formalise dans la proposition D :

1. Nom / appellation / millesime
2. Cepages / assemblage
3. Ce qu'il faut attendre dans le verre
4. Comment le servir / avec quoi
5. Region
6. Pays seulement en detail
